# Project Notes: (S)AGE - Sovereign Agent Governed Experience

**Source:** [https://github.com/l33tdawg/sage](https://github.com/l33tdawg/sage)

### Overview
(S)AGE provides a **consensus-validated institutional memory layer** for multi-agent systems. It moves beyond flat-file or simple vector database approaches, instead implementing an architecture built on distributed ledger primitives (CometBFT) to ensure memory integrity, persistence, and validity across autonomous agents.

### Architectural Patterns for Validity Enforcement
(S)AGE enforces memory integrity through a robust, consensus-driven pipeline, making it a reference implementation for "validity-checking" agent architectures.

*   **Consensus-First Writes:** Memory submissions are not stored immediately. They are treated as transactions that must pass through an ABCI (Application BlockChain Interface) pipeline.
*   **Validator-Driven Validation:** The system utilizes a validator set (e.g., sentinel, deduplication, quality, and consistency checks) that requires a **3/4 or 2/3 BFT quorum** to commit a memory entry.
*   **On-Chain Governance:** Governance decisions (e.g., modifying the validator set, proposing changes) are enacted on-chain, eliminating race conditions. This ensures that agent permissions and system state transitions are deterministic and auditable.
*   **Decay & Persistence:** Unlike static databases, (S)AGE memory incorporates natural decay and confidence scoring, reflecting the lifecycle of institutional knowledge.
*   **Byzantine Fault Tolerance (BFT):** The system integrates Byzantine fault testing directly into the CI/CD pipeline, ensuring that the consensus engine remains resilient even under node failure or malicious input.

### Memory & Knowledge Management
*   **Hybrid Storage:** Combines **PostgreSQL/pgvector** for off-chain semantic storage with **BadgerDB** for on-chain cryptographic hash commitments.
*   **Semantic vs. Keyword Recall:** Employs dual-path retrieval:
    *   **Semantic:** Vector embeddings for intent-based recall.
    *   **Keyword (FTS5):** BM25-ranked full-text search as a fallback when semantic embeddings are insufficient or when the provider is unavailable.
*   **Synaptic Ledger:** Uses **AES-256-GCM encryption** with Argon2id key derivation, ensuring that stored agent knowledge remains secure at rest.

### Agent Reasoning & MCP Integration
*   **Mandatory Boot Sequence:** Enforces a "boot" step (e.g., `sage_inception`) before any agent response, seeding boot compliance as a high-confidence fact in the agent's context.
*   **MCP Governance Tools:** Exposes governance through MCP (Model Context Protocol), allowing agents to propose and vote on system policies, validator sets, or memory constraints directly through standard tool interfaces.
*   **Contextual Hook Enforcement:** Automatically patches and maintains Claude Code hooks (`SessionStart`, `PreCompact`, etc.) to ensure that agents adhere to memory lifecycle protocols across sessions, even after compaction.

### Applicability to Validity-Checking Agents
(S)AGE provides a blueprint for building agents that do not blindly trust input. By requiring **consensus-validation** before memory can be integrated, it forces agents to rely on "Institutional Memory" that is:
1.  **Attributable:** Linked to specific agents and keys.
2.  **Validated:** Passed through a multi-node consensus check for quality and consistency.
3.  **Auditable:** All state transitions and memory writes are recorded in a hash-linked ledger.

### Research Validation
The repository provides empirical backing for the architectural choices:
*   **Calibration Accuracy:** Demonstrated a statistically significant improvement in agent calibration (Cohen's d = -0.860).
*   **Cumulative Learning:** Longitudinal studies show agents with (S)AGE memory achieve superior performance (Spearman rho = 0.716) compared to memoryless baselines.
